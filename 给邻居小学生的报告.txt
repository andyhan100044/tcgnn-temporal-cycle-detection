================================================================================
我用一年时间研究了什么 —— 一个国际贸易专业学生的最新工作
作者：韩轩桐
日期：2026 年 6 月
================================================================================


【开场白：为什么这个研究重要】

我先问一个问题：你有没有想过，洗钱、骗退税这些事情，到底是怎么被发现的？

电视剧里都是警察冲进门抓人。但现实里，坏人玩的"骗术"比电视剧精得多——
他们不是抢银行，而是用一堆看起来完全合法的交易，把"脏钱"洗成"干净钱"。
警察局一年因为这种骗局损失几百亿美元。


【坏人的骗术：三个人递一张钱】

我讲个最简单的骗术。

想象有三个同学：小明、小红、小刚。
他们想做坏事，但手里只有 100 元赃款。

步骤 1：小明 → 小红，假装"卖了一本漫画书"，收了 100 元。
步骤 2：小红 → 小刚，假装"卖了一个玩具"，又收了 100 元。
步骤 3：小刚 → 小明，假装"卖了一个游戏卡"，又收了 100 元。

从账面上看：发生了 3 笔交易，总共 300 元。
但实际上：还是那 100 元，在三个人手里转了一圈。

这种骗术在真实的国际贸易里有个名字叫"空转套利"。
海关看流水账：3 笔合法的进出口贸易，每笔都符合规定，所以放行退税。
坏人就这么骗走了出口退税和跨境资金额度。

真实世界里不是 3 个人，是几十家公司、几十笔交易，时间跨度几个月。
循环越长越复杂，越难发现。


【为什么难抓？两个原因】

原因 1：单看任何一笔交易都是合法的。
　 小明卖给小红漫画书，没毛病。
　 小红卖给小刚玩具，没毛病。
　 单独看每笔都说得通。

原因 2：时间顺序很重要，但人手看不过来。
　 真正的贸易循环一定是"先卖出去 → 钱回来 → 再买进来 → 再卖"
　 每一步时间都不能乱。
　 一个月可能有几十万笔交易，让人盯着看谁的时间顺序绕了一圈，不现实。


【我做了什么：训练了一个"超级侦探"】

我写了一个程序，专门识别"这种绕圈的骗术"。

程序名叫 TC-GNN（Temporal Cycle GNN，翻译过来就是"找时间绕圈的神经网络侦探"）。

它做三件事：

────────────────────────────────────
第一件事：把交易变成"圈圈图"

它会把所有公司画成点，把所有交易画成箭头。
小明 → 小红 是一个箭头，小红 → 小刚 又是箭头...
然后它专门找"箭头绕回起点"的圈圈。
────────────────────────────────────
────────────────────────────────────
第二件事：检查每个圈圈是否"可疑"

不是所有圈圈都是骗钱。
比如 3 个供应商之间互相关照生意，钱也是真的，那是合法的。

所以程序会检查每个圈圈：
  (1) 时间是不是严格按顺序递增？（前后乱跳的圈圈不是洗钱）
  (2) 金额是不是大致守恒？（进出差不多的圈圈可疑）
  (3) 这个圈圈有没有触及"已知有问题的公司"？（触及了就可疑）

这三条规则叫做"硬约束"。
意思是：如果一个圈圈连这三条都不满足，那它 100% 不是洗钱圈。
────────────────────────────────────
────────────────────────────────────
第三件事：用"图神经网络"深度分析

"图神经网络"你可以理解成一个"侦探推理网络"。
它不是只看一笔交易，而是同时看：
  - 这家公司跟谁做生意
  - 它的钱从哪来、到哪去
  - 它历史上有没有可疑行为

这就像老警察办案：他认识的人里，谁跟黑社会有关系、谁信誉不好，
侦探心里都有数。一查就知道哪些公司可疑。
────────────────────────────────────


【为什么这个研究牛？我从三个角度说】

角度 1：理论上牛 —— 我证明了"找洗钱圈"这件事是 NP 完全问题。

"NP 完全问题"是计算机科学里的术语。
意思就是：这个问题非常难，目前没有已知的高效解法。
哪怕用最快的计算机，最坏情况下也要算指数级时间。

我写了一份**严格证明**（不是猜，是数学证明）说：这个问题跟经典的"3-SAT"问题一样难。
这是给学术圈的"硬通货"——证明问题难，本身就是有价值的成果。

角度 2：算法上牛 —— 我设计了两种算法。

  - **精确算法**（TemporalDFS）：把可能的圈圈一个个试，时间长但保证找全。
  - **近似算法**（TemporalCycleSketch）：用一种叫"草图"的数据结构快速筛，
     速度快 100 倍，几乎不漏掉真圈圈，误报率可控。

角度 3：模型上牛 —— 我设计的 TC-GNN 模型在真实数据上打败了所有对比方法。

我用了公开的"比特币反洗钱数据集"（Elliptic1，有 20 万笔交易、4 千多个已知骗子）来测试。
模型做了严格的统计检验（bootstrap 置信区间 + DeLong 检验）。

结果：
  - TC-GNN 优化版：AUC = 0.962（满分 1.0，越高越好）
  - 普通 GNN 基线：AUC ≈ 0.83-0.84
  - 统计显著性：p < 0.001（意思是非常非常显著，不是巧合）

简单说：我的方法比现有最好的方法还准 12 个百分点。


【我也承认了局限性 —— 诚实很重要】

但是！科学不能只报喜不报忧。我也发现了我的方法的弱点。

我用 4 种"现实中的合法圈圈"做压力测试（这些都是真实反洗钱里常见的假阳性）：
  - 退款链：客户付款 → 商户退款（看起来像圈圈，其实是正常退货）
  - 交易所路由：钱经过交易所中转（合法行为）
  - 时间扰动的合法圈：业务周期造成的循环付款
  - 长链近失配：复杂的多方交易

结果：
  - 在"近失配"评估下，TC-GNN 表现完美（AUC 0.96）
  - 但在"现实假阳性"压力测试下，TC-GNN 跌到了 0.53（接近瞎猜）

为什么？
因为我的方法只看"圈圈的形状"，不看"圈圈里的公司是不是坏人"。
这暴露了一个根本问题：**很多合法圈圈也满足时间递增 + 金额守恒**。
要让模型区分"好圈圈"和"坏圈圈"，必须看圈圈里**节点的来源**（provenance）。

我把这个问题写进了论文，并提出了下一步工作 **TC-GNN2**：
未来的版本会结合"形状" + "来源"两种判断。

这才是负责任的研究态度——我告诉你什么能行、什么不能行、为什么不能行、下一步怎么办。


【对比一下：别人怎么做、我的区别是什么】

别人（学术界）：
  - 静态关联规则：只看"这家公司历史上是不是有问题"，不看时间顺序。
  - 普通 GNN：能看到公司的关系，但不用"时间递增"这种业务规则。
  - 手工程序特征（如 XGBoost）：专家手动设计 9 个数字特征，然后丢给模型。
    这种方法在我们的数据上 AUC = 1.000（满分！）——但这是因为我们的负样本
    被人为设计得太容易区分。真实情况没那么容易。

我的 TC-GNN：
  - 第一个把"业务时间约束"和"金额守恒约束"作为**硬约束**嵌入神经网络的。
  - 不只是预测概率，还告诉你"为什么"（哪个圈、哪几个公司、什么时间）。
  - 有数学证明（NP 完全性 + 复杂度分析）支撑，不是黑盒。


【这个研究有什么用？】

短期（学术）：发一篇顶级论文（IEEE TNNLS）。
我的论文准备好了，可以直接投稿。

中期（应用）：帮海关、银行、反洗钱部门自动化筛查。
以后可能集成到：银行的风控系统、支付宝的反洗钱模块、海关的退税审核。

长期（理论）：启发其他领域的"时序闭环检测"。
比如：
  - 社交网络里识别"水军互刷"的小团体
  - 金融供应链里识别"循环担保"
  - 加密货币里识别"混币器洗钱"


【需要什么才能做出能用的产品？】

我现在的研究是"方法验证"（在公开数据集上证明有效）。
要做成真正能部署的产品，还需要：

  1. 拿真实的海关数据（不是比特币）—— 需要海关总署合作
  2. 拿更大的 Elliptic2 数据（49M 节点，88 GB）—— 需要 Kaggle API 或合作
  3. 解决"假阳性"问题 —— TC-GNN2 的研究方向
  4. 多家银行联合部署 —— 联邦学习技术
  5. 法规合规 —— 跟监管机构合作设计部署方案


【总结：我做了什么、为什么值得】

我用 1 年的时间：
  - 把一个 0 人在做的"贸易循环骗税"问题形式化成数学模型问题
  - 证明了它非常难（NP 完全）
  - 设计了 2 种算法（1 个精确 + 1 个近似）
  - 设计了 1 个神经网络（TC-GNN），把业务规则作为硬约束
  - 在 20 万笔真实比特币反洗钱数据上验证：比现有方法好 12%，统计显著
  - 诚实地承认了局限性，并提出了改进方向

这不是一个"灵光一现"的成果。
这是一个**完整的体系**：理论 + 算法 + 模型 + 实验 + 局限性分析 + 未来路线。

我把代码、实验、论文全部开源在 GitHub 上：
https://github.com/andyhan100044/tcgnn-temporal-cycle-detection

任何人可以下载、跑实验、验证结果、改进方法。

【AML 是什么？用洗衣服打比方】

AML = Anti-Money Laundering（反洗钱）。

一句话：把所有"把脏钱洗成干净钱"的办法都堵上。

打个比方：

| 概念 | 比喻 |
|---|---|
| 脏钱 | 一件脏衣服 |
| 洗钱 | 把脏衣服塞进洗衣机，洗到看不出是脏的 |
| 反洗钱（AML） | 在洗衣店门口蹲守的警察 |

为什么 AML 难？因为警察看到的只是"有人买了台洗衣机"（合法行为），
不知道他是要洗脏衣服还是洗正常衣服。
又不能把每个人每笔交易都查一遍（太贵、侵犯隐私）。

所以 AML 的本质是：**在没有搜查令的情况下，从正常人中挑出少数坏人**。


【学术圈 5 派警察查案思路】

研究 AML 的人大致分 5 派，就像警察局有 5 种查案风格：


第 1 派：规则派（最老派，爷爷辈的）

思路：写 if-then-else 规则。
  - 单笔超过 1 万美元？报警。
  - 名字在联合国黑名单上？报警。
  - 一年内向 5 个不同国家汇钱？报警。

比喻：小区的保安大叔——短发纹身的拦下来盘问，西装革履的直接放行。

优点：简单、可解释、银行已经在用。
缺点：骗子学会绕过（拆成 9999 美元一笔；用朋友的账户；去小国转账）。


第 2 派：传统机器学习派

思路：把每笔交易变成数字（金额、频率、国家），扔给模型分类。
模型从历史数据学：什么样的交易"像"洗钱？

比喻：警察看每个人的档案——有没有前科、职业是什么、收入是否合理。综合打分。

优点：能发现复杂模式（不是简单规则能抓住的）。
缺点：只看到单笔交易，**看不到"圈"**——看不到"A→B→C→A"这种循环。


第 3 派：图神经网络派（GNN）← 我们属于这派

思路：把人和公司画成"点"，交易画成"箭头"。看网络结构而不是单笔。

比喻：警察不只看单个人，而是看他的**社交圈**——周围都是什么人？钱的流向是什么样？

这一派 2018 年开始火起来，做的事情是：
  - 用 GNN 看交易网络结构
  - 把"周围都是骗子"的人标为可疑
  - 比单笔看准很多

但 GNN 派有个共同盲点：**只看图结构，没看时间**。
"今天给 B 转 1 万"和"一年前给 B 转 1 万"——对 GNN 来说一样。
但对警察来说完全不一样——一年前的可能是朋友转账，今天的可能就是洗钱。

**我们的工作属于这派的最新子方向**：
  - 不仅看图，还看**时间顺序**
  - 还用**业务规则**（海关申报周期、金额守恒）作为硬约束

换句话说：第 3 派是"看社交圈"，我们是"看社交圈 + 看谁先联系谁 + 看钱是否守恒"。


第 4 派：弱监督/半监督派

思路：承认一个尴尬事实——现实中确认的洗钱案太少了（警察 1 年能破的案子就那么点），
模型训练数据不够用。所以用"看起来像洗钱"的标签代替。

比喻：警察没有嫌疑人名单，但知道"赌博网站的客户常去 ATM 取现金"。
把"赌博网站取款"作为"准嫌疑人"来排查。

我们的论文**主动承认**：我们的环级标签是"业务规则启发式"，**不是 ground truth**。
这正是第 4 派的思路。学术上叫"弱监督"——不装作有完美标签，老实说这是凑合的。


第 5 派：综述派（写总结的）

思路：不研究具体方法，写"过去 5 年大家做了什么"的 review paper。

比喻：警察局长写年度总结——"今年各分局用了哪几种方法，各方法的优缺点"。

我们的论文相关工作这一节引用了 4 篇综述（Lawal 2025、BIS Aurora、中文综述等），
并指出它们的局限：**都停留在"问题陈述"，没深入"时序闭环"这个具体结构**。


【我的工作在 5 派里的位置】

```
                        [看到时间顺序？]
                              │
              ┌───────────────┴────────────────┐
              │                                │
            没有                              有 ← 我们
              │                                │
   ┌──────────┼─────────┐                      │
   │          │         │                      │
规则派     ML派     GNN静态              [看到业务规则？]
（爷爷辈）  (档案)   (社交圈)                  │
                                              │
                                  ┌───────────┴─────────────┐
                                  │                         │
                                没有                       有 ← 我们
                                  │                         │
                          普通GNN文献              我们的TC-GNN
                          (Lawal, BIS等)          (硬约束嵌入)
```

简单说：
  - 第 1-4 派都在"用更好的工具抓坏人"
  - 我们做的事情是"重新定义'坏'长什么样"
  - 在前 4 派的工具基础上，加了一个新维度：**业务时间约束 + 价值守恒约束**


【为什么前人没做？】

不是他们没想到。是：
  - 业务时间约束难形式化（每个国家的海关规则不一样）
  - 价值守恒约束在加密货币里管用，在贸易里反而是错的信号（贸易骗税故意虚增金额）
  - 跨域迁移（加密 → 贸易）有 3 个根本缺陷，论文最后一节专门讨论

我们没解决所有问题。但我们**指出了这些问题，并给出了数学补救方向**。
