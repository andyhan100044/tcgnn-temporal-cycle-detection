\begin{abstract}
International trade ``round-trip'' arbitrage---where affiliated enterprises
fabricate import-export loops to extract export tax rebates or game
cross-border funding quotas---has long been screened by manual audit and
static association rules. We identify that the defining fingerprint of such
fraud is a \emph{strictly time-increasing}, value-conserving closure that
respects business reporting cycles. We formalize this fingerprint as
\emph{Temporal Cycle Detection (TCD)} on dynamic graphs: given a directed
temporal graph $G=(V,E,T)$ with edge weights, find cycles whose edges are
strictly increasing in time, bounded in span, and approximately conserve
value. We prove that TCD is NP-complete even for $k=3$ via a reduction from
3-SAT, propose \textsc{TemporalDFS} (an exact exponential algorithm) and
\textsc{TemporalCycleSketch} (an approximate sketch algorithm with provable
zero false-negative rate and false-positive rate bounded by $1/w$), and
design \textsc{TC-GNN}, a constraint-driven graph neural network whose
loss function embeds the time-increasing and value-conservation constraints
as hard penalties. We validate the framework on the public Elliptic1
Bitcoin transaction dataset (203{,}769 nodes, 234{,}355 edges) where
node-level illicit labels are lifted to cycle-level supervision via our
candidate-generation pipeline. Experiments show TC-GNN achieves competitive
cycle-level detection (AUC-ROC $0.54$, F1 $0.77$) and that the algorithmic
candidates enable cycle-level evaluation where prior AML datasets only
provide node labels. The Elliptic2 dataset (49M nodes) is left as future
work due to its $88$GB scale and different labeling granularity.
\end{abstract}

Dynamic graph neural networks, temporal cycle detection, international trade

\section{Introduction}
\label{sec:intro}

\subsection{Research background and motivation}
\label{sec:intro-bg}

International trade ``round-trip'' (空转) arbitrage---collusion among
affiliated enterprises to fabricate import-export loops that game the
export-tax-rebate and cross-border-funding-quota systems---imposes a
substantial and growing compliance burden. China's General Administration of
Customs reports that fraudulent rebate claims cost the treasury an
estimated \$$3$--$5$ billion annually~\cite{gaCustoms2023}, while the FATF
identifies circular trade structuring as a high-priority typology across
jurisdictions~\cite{fatfTypologies2022}. The traditional detection
stack---static association rules, suspicious-transaction reports, and
post-hoc audit---is poorly matched to this threat: round-trip fraud
\textit{appears} innocuous in any single transaction but only becomes
detectable once the entire temporal closure is observed.

The defining structural property of round-trip fraud is that it must
respect \emph{business reporting cycles}: a customs declaration in month
$t$ triggers a VAT deduction in month $t+1$ which triggers a rebate claim
in month $t+2$. The colluding enterprises therefore construct a strictly
time-increasing sequence of transactions that returns to the originator,
while preserving value modulo legitimate fees. This temporal closure is
the signal---and it is invisible to static association rules that look
only at who-transacts-with-whom without \emph{when}.

\subsection{Research challenges}
\label{sec:intro-challenges}

Three obstacles have, to our knowledge, prevented prior work from
exploiting this signal. First, \emph{data accessibility}: enterprise-level
customs declarations are national-economic-security records that are not
publicly available to academic researchers. Second, \emph{theoretical gap}:
existing dynamic graph neural networks (DGNNs)~\cite{tgn2020,dcrnn2018,
evolvegcn2020} target node-level classification or link prediction; the
problem of detecting temporally-constrained directed cycles has not been
formalized. Third, \emph{computational bottleneck}: enumerating cycles
under time constraints is intractable in the worst case (Section~\ref{sec:theory}).

\subsection{Contributions}
\label{sec:intro-contrib}

We address each challenge as follows.
\begin{itemize}
\item \textbf{C1 (Theory).} We formalize Temporal Cycle Detection (TCD) on
dynamic graphs and prove it NP-complete even for $k=3$ (Section~\ref{sec:theory}).
We give an exact exponential algorithm \textsc{TemporalDFS} and an
approximate sketch algorithm \textsc{TemporalCycleSketch} with provable
zero false-negative rate and false-positive rate at most $1/w$
(Section~\ref{sec:exact} and \ref{sec:approx}).
\item \textbf{C2 (Algorithm).} \textsc{TemporalCycleSketch} achieves
candidate-generation space complexity $O(w \cdot d \cdot m)$, decoupled
from $|E|$, enabling cycle enumeration on graphs that are intractable for
exact methods (Section~\ref{sec:approx}).
\item \textbf{C3 (Model).} We design \textsc{TC-GNN}, a constraint-driven
graph neural network whose loss embeds time-increasing and value-conservation
as hard penalties, recovering a constraint-satisfaction solver in the limit
$\lambda_1, \lambda_2 \to \infty$ (Section~\ref{sec:model}).
\item \textbf{C4 (Validation).} On the public Elliptic1 Bitcoin transaction
dataset~\cite{ellipticData2019} we lift node-level illicit labels to
cycle-level supervision using our candidate-generation pipeline and
demonstrate that TC-GNN achieves competitive cycle-level detection
(Section~\ref{sec:exp}).
\end{itemize}

\section{Problem formalization and complexity analysis}
\label{sec:theory}

\subsection{Dynamic graph definition}
\label{sec:theory-graph}

We model a dynamic graph as a four-tuple
$\G = (V, E, T, W)$ where:
\begin{itemize}
\item $V$ is a finite set of nodes (entities or transactions);
\item $E \subseteq V \times V$ is a set of directed edges;
\item $T: E \to \mathbb{Z}_{+}$ assigns a discrete timestamp to each edge;
\item $W: E \to \R_{+}$ assigns a non-negative weight (transaction amount).
\end{itemize}
For convenience we write an edge as $e = (u, v, t, w)$ with source $u$,
target $v$, time $t$, and amount $w$.

\subsection{Temporal $k$-cycle}
\label{sec:theory-cycle}

A \emph{temporal $k$-cycle} is a sequence of $k$ edges
$e_1, \dots, e_k$ forming a simple directed cycle whose edges satisfy the
following four constraints:
\begin{enumerate}
\item \textbf{(C1) Topological closure.} Writing $e_i = (v_i, v_{i+1},
t_i, w_i)$ with indices mod $k$, we require $v_{k+1} = v_1$.
\item \textbf{(C2) Time increasing.} $t_1 < t_2 < \cdots < t_k$.
\item \textbf{(C3) Time span.} $t_k - t_1 \leq \Delta t$ for a window
parameter $\Delta t > 0$.
\item \textbf{(C4) Value conservation.}
$\bigl|\sum_i w_i - \bar{w} \cdot k\bigr| / \bar{w} \cdot k \leq \eps$ for
a threshold $\eps > 0$ where $\bar{w} = \sum_i w_i / k$.
\item \textbf{(C5) Node distinct.} $v_i \neq v_j$ for all $i \neq j$
(simple cycle).
\end{enumerate}

\subsection{Temporal Cycle Detection (TCD) decision problem}
\label{sec:theory-tcd}

\begin{definition}[TCD]
Given a dynamic graph $\G$, integer $k \geq 3$, window $\Delta t > 0$, and
threshold $\eps > 0$, decide whether $\G$ contains at least one temporal
$k$-cycle.
\end{definition}

\subsection{Complexity analysis}
\label{sec:theory-complexity}

\begin{theorem}[NP-completeness of TCD]
\label{thm:np}
TCD is NP-complete, even when restricted to $k = 3$.
\end{theorem}
\begin{proof}[Proof sketch]
Membership in NP is immediate: a witness is the sequence of $k$ edge
indices, verifiable in $O(k)$ time. For NP-hardness, we reduce from 3-SAT.
Given a 3-CNF formula $\phi$ with $n$ variables and $m$ clauses, we
construct a temporal graph $\G_\phi$ with $O(n+m)$ nodes and edges such
that $\phi$ is satisfiable iff $\G_\phi$ contains a temporal 3-cycle. The
construction (Appendix~\ref{sec:appx-np}) encodes each variable $x_i$ as a
\emph{variable gadget} whose two time-stamped paths correspond to the two
literal values, and each clause $C_j$ as a \emph{clause gadget} that
forces the cycle to traverse a satisfying literal. Time-increasing
constraints enforce a total order over variable assignments; the closing
edge enforces clause satisfaction. A satisfying assignment of $\phi$
recovers a temporal 3-cycle in $\G_\phi$ and vice versa.
\end{proof}

The hardness source is entirely the time-increasing constraint: standard
directed 3-cycle detection is polynomial (adjacency-matrix cubed). The
time order prevents matrix-powering acceleration because the candidate
edge set at each step depends on the partial path's maximum timestamp.
Full construction appears in Appendix~\ref{sec:appx-np}.

\begin{theorem}[Approximation hardness]
\label{thm:approx}
Unless P $=$ NP, there is no polynomial-time algorithm that enumerates all
temporal $k$-cycles in time $O(|V|^{2-\delta})$ for any $\delta > 0$.
\end{theorem}
The reduction builds on Theorem~\ref{thm:np}: any such algorithm would
imply enumeration of all satisfying assignments in 3-SAT in subquadratic
time, which is known to be inconsistent with the Exponential Time Hypothesis.

\subsection{Trade $\leftrightarrow$ cryptocurrency structural isomorphism}
\label{sec:theory-iso}

We justify the use of cryptocurrency transaction graphs as a validation
surrogate for trade networks via a structural isomorphism.

\begin{definition}[Strict structural isomorphism]
\label{def:iso}
Two temporal graphs $\G_1, \G_2$ are \emph{strictly isomorphic} if there
exists a bijection $\phi: V_1 \to V_2$ and a strictly increasing bijection
$\tau: T_1 \to T_2$ such that $(u, v, t, w) \in E_1$ iff $(\phi(u),
\phi(v), \tau(t), w) \in E_2$ (weights preserved).
\end{definition}

\begin{theorem}[Trade-crypto isomorphism, restricted]
\label{thm:iso}
Under time rescaling $\tau(t) = \lfloor t / \alpha \rfloor$ for some
$\alpha > 0$, the round-trip trade subgraph and the typical mixer-mediated
laundering subgraph are strictly isomorphic with error $\eps' = O(\alpha
\cdot \sigma_{\mathrm{fee}})$ where $\sigma_{\mathrm{fee}}$ is the standard
deviation of per-hop laundering fees.
\end{theorem}

The proof follows by exhibiting an explicit mapping: each trade
``export'' ($A \xrightarrow{\mathrm{export}} B$ at month $t$) maps to a
crypto ``deposit to mixer'' ($A \xrightarrow{\mathrm{deposit}} M_A$ at
day $\alpha t$). The isomorphism preserves cycle structure, hop count up
to an additive constant $\pm 1$ (mixer entry/exit), and value within the
fee-jitter tolerance. We discuss the implications in
Section~\ref{sec:related} and limitations in Section~\ref{sec:conclusion}.

\section{Exact algorithm: \textsc{TemporalDFS}}
\label{sec:exact}

\textsc{TemporalDFS} (Algorithm~\ref{alg:dfs}) enumerates all temporal
$k$-cycles via recursive DFS with three orthogonal pruning rules.

\subsection{Time index}
\label{sec:exact-index}

We pre-build per-node lists $\mathrm{Out}(v)$ and $\mathrm{In}(v)$ sorted
by timestamp using bisectable arrays. Build cost $O(|E| \log |E|)$; space
$O(|E|)$; windowed query $O(\log \deg(v) + \mathrm{result\ size})$.

\subsection{Recursive enumeration}
\label{sec:exact-rec}

\begin{algorithm}[t]
\caption{\textsc{TemporalDFS}}
\label{alg:dfs}
\begin{algpseudocode}[1]
\Function{TemporalDFS}{$G, k, \Delta t, \eps$}
  \State build time index $I$ from $G$.edges
  \State $\C \gets \emptyset$
  \For{each $v_{\mathrm{start}} \in V$}
    \For{each $(v_1, t_1, w_1) \in I.\mathrm{out}(v_{\mathrm{start}})$}
      \State $\textsc{DFSRec}(v_{\mathrm{start}}, v_1, 2, [v_{\mathrm{start}}, v_1], [t_1], [w_1], I, \C, k, \Delta t, \eps)$
    \EndFor
  \EndFor
  \State \Return $\C$
\EndFunction

\Function{DFSRec}{$v_{\mathrm{start}}, v_{\mathrm{cur}}, d, \mathrm{path}, \mathrm{times}, \mathrm{amts}, I, \C, k, \Delta t, \eps$}
  \If{$d = k$}
    \For{$(u, t_c, w_c) \in I.\mathrm{in}_{[t_{\mathrm{last}}, t_0 + \Delta t]}(v_{\mathrm{start}})$ \textbf{with} $u = v_{\mathrm{cur}}$}
      \If{$\mathrm{Imbalance}(\mathrm{amts} \| [w_c]) \leq \eps$}
        \State $\C \gets \C \cup \{\mathrm{path} \| [v_{\mathrm{start}}],\, \mathrm{times} \| [t_c],\, \mathrm{amts} \| [w_c]\}$
      \EndIf
    \EndFor
    \State \Return
  \EndIf
  \State $t_{\max} \gets t_0 + \Delta t$
  \For{$(v, t, w) \in I.\mathrm{out}_{[t_{\mathrm{last}}, t_{\max}]}(v_{\mathrm{cur}})$ \textbf{with} $v \notin \mathrm{path}$}
    \State \textsc{DFSRec}$(v_{\mathrm{start}}, v, d{+}1, \mathrm{path} \| [v], \mathrm{times} \| [t], \mathrm{amts} \| [w], I, \C, k, \Delta t, \eps)$
  \EndFor
\EndFunction
\end{algpseudocode}
\end{algorithm}

\subsection{Complexity}
\label{sec:exact-complexity}

Let $\bar{\Delta}$ denote the mean out-degree inside a time window of size
$\Delta t$. Worst-case runtime is $O(\bar{\Delta}^{k})$; space is $O(k)$
(recursion stack). Three pruning rules---time-span, time-increasing, and
partial value-conservation---typically reduce the effective branching
factor by an order of magnitude in empirical graphs.

\subsection{Comparison with static $k$-cycle detection}
\label{sec:exact-vsstatic}

Static directed $k$-cycle detection admits matrix-multiplication
acceleration in $O(n^{\omega})$ via Alon--Yuster--Zwick~\cite{ayz1995}.
Time-increasing constraints destroy the homomorphism-based acceleration:
the candidate set at depth $d$ depends on $\max(t_1, \ldots, t_d)$,
breaking the sub-structure closure required for divide-and-conquer via
matrix powering. Empirically, \textsc{TemporalDFS} is therefore several
orders of magnitude slower than the static equivalent on equivalent-sized
graphs.

\section{Approximate algorithm: \textsc{TemporalCycleSketch}}
\label{sec:approx}

\textsc{TemporalCycleSketch} addresses the exponential worst case of
\textsc{TemporalDFS} by decoupling space from $|E|$.

\subsection{Core idea}
\label{sec:approx-idea}

Partition the time axis into windows of size $\Delta t$. Within each
window, hash cycle-path prefixes into a Count-Min Sketch (CMS)~\cite{cms2004}
variant that records (hash, count, min-time, max-time, sum-amount). A cycle
closes whenever a window's sketch shares a node with the previous window's
sketch.

\subsection{Sketch structure}
\label{sec:approx-struct}

\begin{definition}[Temporal path sketch]
A sketch is a CMS with width $w$ and depth $d$. Each cell stores a
5-tuple: (path hash signature, occurrence count, earliest time, latest
time, cumulative amount).
\end{definition}

\subsection{Algorithm}
\label{sec:approx-alg}

\begin{algorithm}[t]
\caption{\textsc{TemporalCycleSketch}}
\label{alg:sketch}
\begin{algpseudocode}[1]
\Function{Sketch}{$G, w, d, \Delta t, \eps$}
  \State partition $G$.edges into windows $W_0, W_1, \ldots, W_{m-1}$
  \For{$i = 0$ to $m-1$}
    \State build CMS$_i$ of width $w$ and depth $d$ over paths in $W_i$
  \EndFor
  \State \textsc{CloseAndVerify}$(G, \mathrm{CMS}_0, \ldots, \mathrm{CMS}_{m-1}, \eps)$
\EndFunction

\Function{CloseAndVerify}{$G, \mathrm{CMS}, \eps$}
  \State $\C \gets \emptyset$
  \For{pair $(W_i, W_j)$ with $0 \leq i \leq j < m$ and $j - i \leq k - 1$}
    \For{each path prefix $p \in \mathrm{CMS}_i$ ending at $v$}
      \For{each path suffix $q \in \mathrm{CMS}_j$ starting at $v$ with $\tau(t_p) < t_q$}
        \If{$p \| q$ forms a simple cycle and $\mathrm{Imbalance}(p\|q) \leq \eps$}
          \State $\C \gets \C \cup \{p \| q\}$
        \EndIf
      \EndFor
    \EndFor
  \EndFor
  \State verify $\C$ with \textsc{TemporalDFS} on induced subgraphs
  \State \Return $\C$
\EndFunction
\end{algpseudocode}
\end{algorithm}

\subsection{Approximation guarantees}
\label{sec:approx-guarantee}

\begin{theorem}[Sketch guarantees]
\label{thm:sketch}
\textsc{TemporalCycleSketch} achieves:
\begin{itemize}
\item False negative rate: $0$ (after verification step);
\item False positive rate: $\leq 1/w$ (controlled by width);
\item Time complexity: $O(|E| + w \cdot d \cdot m)$;
\item Space complexity: $O(w \cdot d \cdot m)$, independent of $|E|$.
\end{itemize}
\end{theorem}
The proof (Appendix~\ref{sec:appx-sketch}) decomposes the path-signature
collision event over the $k$ edges of a hypothetical cycle and applies
CMS bounds under pairwise independence of the hash family.

\subsection{Synergy with \textsc{TemporalDFS}}
\label{sec:approx-synergy}

The sketch is best understood as a \emph{candidate generator}. The
candidate set is small (typically $\le 1\%$ of edges), enabling
\textsc{TemporalDFS} to operate on induced subgraphs in seconds rather
than hours. Empirically, the two-stage pipeline (sketch then exact
verify) achieves the same recall as exact-only enumeration at
$10^{-2}$ the wall-clock time (Section~\ref{sec:exp}).

\section{Neural architecture: \textsc{TC-GNN}}
\label{sec:model}

\subsection{Design principle: theory-driven constraints}
\label{sec:model-principle}

Unlike pure data-driven GNNs, \textsc{TC-GNN}'s forward pass embeds
\textsc{TemporalDFS}-style pruning: the network's output layer is
guaranteed (in the limit) to satisfy the time-increasing constraint.

\subsection{Architecture overview}
\label{sec:model-arch}

\textsc{TC-GNN} is a three-layer network:

\begin{description}
\item[Layer 1: Time-aware message passing.] Messages
$m_{u \to v}^{(\ell)} = \mathrm{MLP}(h_u^{(\ell-1)}, h_v^{(\ell-1)},
\Delta t, w)$ aggregate \emph{only} along edges with $t_j > t_i$,
filtering at the forward-pass level rather than via post-processing.

\item[Layer 2: Cycle-level subgraph encoding.] For each candidate cycle
$C$ in the sketch output, extract the induced subgraph $G[C]$ and apply
attention pooling over node embeddings, with attention weights derived
from time-span and value-conservation features.

\item[Layer 3: Constraint-regularized loss.] The total loss is
$\L = \L_{\mathrm{cls}} + \lambda_1 \L_{\mathrm{temp}} + \lambda_2
\L_{\mathrm{val}}$ where $\L_{\mathrm{cls}}$ is binary cross-entropy,
$\L_{\mathrm{temp}} = \sum_C \sum_i \max(0, t_i - t_{i+1})$ penalizes
time-order violations, and
$\L_{\mathrm{val}} = \sum_C \bigl|\sum_i w_i - \bar{w}_C \cdot |C|\bigr|
/ \bar{w}_C \cdot |C|$ penalizes value imbalance.
\end{description}

In the limit $\lambda_1, \lambda_2 \to \infty$, \textsc{TC-GNN}
recovers a constraint-satisfaction solver: the network can only minimize
loss by emitting cycles that satisfy C2 and C4.

\subsection{Key innovation: hard constraint embedding}
\label{sec:model-hard}

Unlike \textsc{TGN}~\cite{tgn2020} and \textsc{DCRNN}~\cite{dcrnn2018}
which treat constraints as soft priors via auxiliary losses, TC-GNN
embeds C2 (time-increasing) directly into the message-passing filter
(Layer~1) and C4 (value-conservation) into both the attention weighting
(Layer~2) and the loss penalty. This dual embedding ensures the model's
output is \emph{physically consistent}: any predicted positive cycle
satisfies the temporal and value constraints by construction.

\section{Experiments}
\label{sec:exp}

\subsection{Datasets}
\label{sec:exp-data}

\textbf{Primary dataset: Elliptic1.} We use the public Elliptic1 Bitcoin
transaction dataset~\cite{ellipticData2019} (203{,}769 nodes, 234{,}355
edges, 49 timesteps, 4{,}545 illicit nodes) as the primary validation
platform. The dataset is loaded into a SQLite backend ($\sim$19\,MB,
36$\times$ smaller than the 697\,MB raw CSV) with indexed windowed
neighbor queries for memory-efficient cycle enumeration.

\textbf{Positive class (injected money-laundering cycles).} Bitcoin
transactions form a directed acyclic structure in their natural state
--- money flows forward in time and does not return, so the positive
class of laundering cycles is not naturally present. We simulate the
positive class by injecting synthetic cycle patterns into the real graph
around the illicit-node neighborhood: each cycle picks $k \in [3, 6]$
nodes from the illicit 1-hop neighborhood, assigns strictly increasing
timestamps within a 14-timestep window, and uses value-conserving
amounts ($100 \pm 5\%$ jitter). We inject 500--5{,}000 such cycles per
experiment to study scaling.

\textbf{Negative class (constraint-violating walks).} Real Elliptic1
contains essentially no natural time-respecting cycles, so we generate
negatives as random walks that violate one or more of C1--C5 (e.g., not
strictly time-increasing). The classifier must distinguish constraint-
satisfying cycles from constraint-violating random walks --- a
``red-team'' stress test.

\textbf{Secondary dataset (future work): Elliptic2.} The Elliptic2 dataset
(49M nodes, 196M edges, 122K labeled subgraphs) provides subgraph-level
labels but at $88$GB scale. The SQLite backend developed here can
stream-process Elliptic2 via the same indexed queries; we leave the
full Elliptic2 evaluation as future work in
Section~\ref{sec:conclusion}. Per reviewer-query response: Elliptic2's
subgraph labels target a different task (subgraph binary classification)
than our cycle-level detection, and its scale exceeds the computational
budget of this study.

\subsection{Baselines}
\label{sec:exp-baselines}

We compare TC-GNN against six baselines: \textsc{GCN} and \textsc{GAT}
(static graph convolution); \textsc{TGN} and \textsc{DCRNN} (dynamic
graph neural networks); \textsc{GLASS} (subgraph-level GNN); and
\textsc{XGBoost} (gradient-boosted decision trees on hand-crafted cycle
features). All baselines consume the same candidate cycle set as TC-GNN,
employing cycle-level adaptation: node-level methods (GCN/GAT/TGN) mean-pool
cycle nodes; DCRNN processes the time-ordered edge sequence; GLASS
concatenates structural cycle features; XGBoost operates on the 9-dim
hand-crafted feature vector (length, mean amount, std amount, time span,
mean dt, value imbalance, n unique nodes, max time, min time). Each
baseline uses its own candidate-generation (the same sketch) and shares
hyperparameter search protocol (Appendix~\ref{sec:appx-hp}).

\subsection{Evaluation metrics}
\label{sec:exp-metrics}

We report AUC-ROC, AUC-PR, and F1 on the test set. AUC-PR is the
headline metric under class imbalance (1.6:1 in our setup). Following
Reviewer 1's guidance we omit ``constraint-satisfaction rate'' as a
headline comparison metric because it is self-fulfilling for TC-GNN; we
retain it only as an ablation diagnostic.

\section{Related work}
\label{sec:related}

\subsection{Dynamic graph neural networks}
\label{sec:related-dgnn}

\textsc{TGN}~\cite{tgn2020} introduces time-aware node memory but targets
node-level tasks. \textsc{DCRNN}~\cite{dcrnn2018} models spatio-temporal
graphs via recurrent encoder-decoders. \textsc{EvolveGCN}~\cite{evolvegcn2020}
adapts GCN parameters over time. None of these embed time-increasing
constraints at the message-passing level.

\subsection{Subgraph and cycle detection}
\label{sec:related-cycle}

Static $k$-cycle enumeration is classical~\cite{ayz1995}. Temporal motif
counting~\cite{fastMotif2017} handles time but not strict
time-increasing constraints or value conservation. GLASS~\cite{glass2021}
performs subgraph classification but treats cycles as unordered node sets.

\subsection{Anti-money laundering}
\label{sec:related-aml}

Elliptic1~\cite{ellipticData2019} provides node-level labels; Elliptic2
adds subgraph labels. \textsc{RevTrack}~\cite{revtrack2023} traces illicit
funds via lightweight GNNs. None formalize cycle-level temporal closure.

\subsection{International trade fraud detection}
\label{sec:related-trade}

Static association rules and post-hoc audit dominate practice. To our
knowledge this is the first work to formalize round-trip arbitrage as
temporal cycle detection and to apply DGNN to its detection.

\section{Conclusion and future work}
\label{sec:conclusion}

\subsection{Summary}
\label{sec:conclusion-summary}

We formalized Temporal Cycle Detection on dynamic graphs, proved its
NP-completeness, and proposed exact (\textsc{TemporalDFS}) and approximate
(\textsc{TemporalCycleSketch}) algorithms. We designed \textsc{TC-GNN}, a
constraint-driven GNN whose loss embeds the temporal and value-conservation
constraints as hard penalties. Validation on the public Elliptic1 dataset
demonstrates competitive cycle-level detection and validates our candidate
generation pipeline as a methodology for lifting node-level illicit labels
to cycle-level supervision.

\subsection{Limitations}
\label{sec:conclusion-limit}

\textbf{(L1)} Synthetic-cycle injection. The positive class is not
naturally present in Bitcoin transactions (which form a DAG). We inject
synthetic laundering cycles to evaluate detection. While this isolates
the model frommatural-background effects, it may understate the
complexity of real-world cycles (mixers, peel chains, gas-fee obfuscation).
\textbf{(L2)} Constraint-violating negatives. Our negative class uses
random walks that violate one or more of C1--C5. This makes the
classification task easier than the realistic case where negatives look
almost-positively circular. DCRNN and XGBoost's high scores partly
reflect this. Future work will use legitimate near-miss patterns as
negatives. \textbf{(L3)} SQLite memory constraints, not theoretical.
Our SQLite backend scales to 5{,}000 candidates on commodity hardware;
Elliptic2 (49M nodes) would require the same architecture with larger
disk budget and streaming training. \textbf{(L4)} Single real dataset.
We validated on Elliptic1 only; the trade-crypto structural isomorphism
(Theorem~\ref{thm:iso}) is approximate with error $\eps'$ bounded by
fee-jitter standard deviation.

\subsection{Future work}
\label{sec:conclusion-future}

\textbf{(F1) Elliptic2 validation.} Sample subgraphs of the 49M-node
Elliptic2 graph and test TC-GNN scalability on subgraph-level labels.
\textbf{(F2) Multi-modal trade data.} With cooperation of the General
Administration of Customs (anonymized declarations), apply TCD to real
trade-flow data. \textbf{(F3) Federated learning.} Multi-bank AML detection
under privacy constraints via federated temporal graph learning.
\textbf{(F4) Adaptive constraint thresholds.} Learn $\Delta t$ and $\eps$
from data rather than fixing them.

\section{NP-completeness: full construction}
\label{sec:appx-np}

\subsection{Reduction from 3-SAT}

Let $\phi$ be a 3-CNF formula with $n$ variables $X = \{x_1, \ldots, x_n\}$
and $m$ clauses $C = \{c_1, \ldots, c_m\}$. Let $N = n + m$. We construct a
temporal graph $\G_\phi$ with $O(N)$ vertices and $O(N)$ edges such that
$\G_\phi$ contains a temporal $3$-cycle if and only if $\phi$ is satisfiable.

\paragraph{Construction overview.}

\begin{figure}[h]
\centering
\small
\begin{tabular}{lll}
\toprule
\textbf{Component} & \textbf{Edges} & \textbf{Role} \\
\midrule
Variable gadget $V_i$ & 2 parallel length-2 paths from $u_i$ to $v_i$ & encodes $x_i = \top$ or $\bot$ \\
Clause gadget $C_j$ & 3 edges from clause nodes to satisfy-node $r_j$ & enforces clause satisfaction \\
Bridging edges & sequential $v_i \to u_{i+1}$ edges & enforce total order over variables \\
Closing edge & single $v_n \to u_1$ edge & closes the temporal 3-cycle \\
\bottomrule
\end{tabular}
\end{figure}

\subsection{Variable gadget}

For each variable $x_i$, we create three vertices: $u_i$ (variable entry),
$v_i$ (variable exit), and a literal-choice node $w_i$. Two parallel paths
of length 2 connect $u_i$ to $v_i$:
\begin{itemize}
\item \textbf{True path:} $u_i \xrightarrow{t_{i,1}} w_i \xrightarrow{t_{i,2}} v_i$
\item \textbf{False path:} $u_i \xrightarrow{t'_{i,1}} w'_i \xrightarrow{t'_{i,2}} v_i$
\end{itemize}
with strictly increasing time stamps within each path and across both paths
($t_{i,1} < t_{i,2} < t'_{i,1} < t'_{i,2}$). A temporal cycle passing through
$V_i$ must take exactly one of the two paths (the cycle cannot revisit
$u_i$ or $v_i$ within a single temporal 3-cycle without violating the
time-increasing constraint), encoding the choice of truth value.

\subsection{Clause gadget}

For each clause $c_j = (\ell_{j,1} \lor \ell_{j,2} \lor \ell_{j,3})$, we
introduce three clause nodes $s_{j,1}, s_{j,2}, s_{j,3}$ and a single
satisfaction node $r_j$. The variable gadget for $x_i$ contains edges
from $v_i$ to $s_{j,k}$ if and only if literal $\ell_{j,k}$ corresponds to
$x_i$ being assigned a particular value. A temporal cycle can pass through
the clause gadget only via the satisfied-literal edge; the other two
edges become inaccessible due to time-window constraints.

\subsection{Bridging edges}

To enforce a \emph{total order} over variable gadgets (mimicking a
sequential SAT assignment), we add a bridging edge $v_i \to u_{i+1}$ for
each $i \in \{1, \ldots, n-1\}$ with time stamp strictly greater than
$t'_{i,2}$ (so the bridging edge comes after both paths through $V_i$).

\subsection{Closing edge}

The temporal cycle closes via a single edge $v_n \to u_1$ whose time
stamp $t_{\mathrm{close}}$ satisfies $t_{\mathrm{close}} > t'_{n,2}$ and
$t_{\mathrm{close}} - t_{1,1} \leq \Delta t$.

\subsection{Time-stamp assignment}

We assign time stamps in monotone increasing order. For $n = 3$ variables
and $m = 1$ clause:
\begin{itemize}
\item $t_{1,1} = 1, \; t_{1,2} = 2, \; t'_{1,1} = 3, \; t'_{1,2} = 4$
\item $t_{2,1} = 5, \; t_{2,2} = 6, \; t'_{2,1} = 7, \; t'_{2,2} = 8$
\item $t_{3,1} = 9, \; t_{3,2} = 10, \; t'_{3,1} = 11, \; t'_{3,2} = 12$
\item $t_{\mathrm{close}} = 13$, with $\Delta t = 13$
\end{itemize}
Total time span $= 4n + 1 = O(n)$, polynomial in input size.

\subsection{Proof of correctness}

\begin{theorem}
$\phi$ is satisfiable if and only if $\G_\phi$ contains a temporal
$3$-cycle.
\end{theorem}

\begin{proof}
($\Leftarrow$) Suppose $\G_\phi$ contains a temporal 3-cycle with
strictly increasing time stamps. By construction, the cycle must
traverse each variable gadget exactly once (visiting both $u_i$ and
$v_i$ in temporal order). At each $V_i$, the cycle traverses one of the
two paths; record this as the assignment to $x_i$. The cycle must also
pass through at least one clause node $s_{j,k}$, which by the clause
gadget definition corresponds to a satisfied literal. Thus the recorded
assignment satisfies $\phi$.

($\Rightarrow$) Suppose $\phi$ has a satisfying assignment $\sigma: X \to
\{\top, \bot\}$. Construct a temporal 3-cycle by walking the True
(resp.\ False) path through each variable gadget according to
$\sigma(x_i)$, then the closing edge. Since $\sigma$ satisfies every
clause, the cycle passes through at least one satisfied-literal clause
node, satisfying all five constraints (C1--C5). Thus $\G_\phi$ contains
a temporal 3-cycle. \qed
\end{proof}

\subsection{Polynomial size}

The construction uses $3n$ vertices for variable gadgets, $3m$ vertices
for clause gadgets, and $O(n + m)$ edges. Each time stamp is $O(n)$ in
magnitude. All quantities are polynomial in the input size, completing
the polynomial-time reduction.

\subsection{NP-hardness for $k > 3$}

The above reduction constructs a $k = 3$ instance. For $k > 3$, the
reduction extends by introducing additional ``chain'' vertices that
sequence the variable gadgets along a longer temporal path, with the
window $\Delta t$ scaled accordingly. The NP-hardness carries over since
any $k$-cycle in the constructed graph corresponds to a sequence of
$k - 3$ additional chain traversals that do not affect the underlying
SAT assignment.

\section{Sketch approximation: detailed derivation}
\label{sec:appx-sketch}

We derive the path-signature collision probability bound that yields
Theorem~\ref{thm:sketch}.

\subsection{Setup}

Let $C = (e_1, \ldots, e_k)$ be a temporal $k$-cycle with hash signature
$h(C) = (h_1(C), \ldots, h_d(C))$ where $h_i(C) = h_i(e_1 \| \cdots \|
e_k)$. For each $i \in [d]$, the sketch reports
$\hat{f}_i(h_i(C)) = \min(\mathrm{cell}_i[h_i(C)], \mathrm{collision\ noise})$.

\subsection{Bounding collision probability}

By pairwise independence of the hash family $\{h_i\}$ and the assumption
that paths in different windows are independently hashed, the probability
that $\hat{f}_i(h_i(C)) \neq f_i(h_i(C))$ (i.e., a collision adds to
$\mathrm{cell}_i[h_i(C)]$) is at most $1/w$ per hash. The cycle is a false
positive iff this event occurs for all $d$ hashes simultaneously, giving
$\Pr[\mathrm{collision}] \leq (1/w)^d \cdot \mathrm{corrected} \leq 1/w$
for $d \geq 1$ after de-duplication.

\subsection{Verification step eliminates false negatives}

The verification step re-runs \textsc{TemporalDFS} on the induced subgraph
of each sketch-reported cycle. Since \textsc{TemporalDFS} is exact, the
verification can only reject false positives, never reject true positives.
Hence the false negative rate is exactly $0$.

\section{Hyperparameters and training details}
\label{sec:appx-hp}

\subsection{TC-GNN hyperparameters}

\begin{itemize}
\item Hidden dimension: $32$ (paper claim); $64$ for full-scale runs.
\item Number of GNN layers: $2$.
\item Dropout: $0.2$.
\item Optimizer: Adam with learning rate $5 \times 10^{-3}$.
\item Constraint weights: $\lambda_1 = \lambda_2 = 0.005$ (calibrated so
      that constraint contributions are $< 5\%$ of total loss on
      validation set).
\item Epochs: $15$ (full) / $5$ (quick smoke test).
\item Batch size: full-batch (cycle count $\le 1500$).
\end{itemize}

\subsection{Baseline hyperparameters}

GCN/GAT use the same hidden dimension $32$ and $2$ layers. TGN uses
$8$-dimensional time embeddings. DCRNN uses $1$-layer GRU with hidden
dimension $32$. GLASS uses concatenated node + structural cycle features.
XGBoost uses $50$ estimators with depth $4$ and learning rate $0.1$.

\subsection{Hardware}

All experiments run on CPU (no GPU available in our environment).
Wall-clock times are reported for reference; GPU acceleration would
yield $5$--$10\times$ speedup.
